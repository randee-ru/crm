# Урок 12 - Как работает API-клиент во frontend

## Простое объяснение

API-клиент - это небольшой слой кода, который знает, как обращаться к backend.
Вместо повторения URL'ов и fetch-кода по всему проекту мы держим это в одном месте.

## Зачем это нужно

Если запросы размазаны по разным страницам, их трудно менять и тестировать.
Один клиент проще поддерживать.

## Что сейчас есть в CRM Kit

- `getHealthcheck()` - пример запроса к `/health/`
- `getClients()` - список клиентов компании из `/api/v1/clients/`
- `getCompanyContext()` - tenant-контекст из `/api/v1/company/`
- `getApiBaseUrl()` и `getCompanySlug()` - настройки подключения
- `frontend/.env.example` - пример переменных окружения для API

## Что изучить новичку

- что такое `fetch`
