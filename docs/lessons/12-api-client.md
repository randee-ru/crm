# Урок 12 - Как работает API-клиент на frontend

## Простое объяснение

API-клиент - это небольшой слой кода, который знает, как обращаться к backend.
Вместо повторения URL'ов и fetch-кода по всему проекту мы держим это в одном месте.

## Зачем это нужно

Если запросы размазаны по разным страницам, их трудно менять и тестировать.
Один клиент проще поддерживать.

## Что сейчас есть в CRM Kit

- `getHealthcheck()` - пример запроса к `/health/`
- `getApiBaseUrl()` - чтение базового URL backend
- `frontend/.env.example` - пример настройки адреса backend

## Что изучить новичку

- что такое `fetch`
- что такое `base URL`
- почему сетевую логику лучше выносить в `lib`
- почему сначала делают простой healthcheck, а потом сложные запросы
