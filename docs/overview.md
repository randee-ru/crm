# Обзор CRM Kit

## Продукт

CRM Kit — облачная CRM/ERP-платформа для сервисного бизнеса.
Первый целевой рынок — **фитнес-клубы**; архитектура рассчитана на расширение в салоны, клиники, SPA, образовательные центры и другой сервисный бизнес.

## Что уже работает

- **Multi-tenant SaaS** — компании, филиалы, роли пользователей
- **Клиенты** — список, поиск, CRUD, блокировки и привязка к абонементам
- **Абонементы** — рабочий CRUD со списком и карточкой
- **Бронирования** — рабочий CRUD со списком и карточкой
- **Посещаемость** — рабочий CRUD и график посещаемости
- **Тренеры** — профили, аренда, доступы и фото
- **CRM** — задачи, **канбан сделок с настраиваемыми воронками**
- **Продажи и платежи** — журналы и CRUD API
- **Расписание** — события, занятия и публичный embed
- **Отчёты, автоматизация, уведомления, интеграции** — backend-каркас и рабочие экраны
- **Frontend** — dashboard в стиле Bitrix24, авторизация, профиль и доменные экраны
- **API** — REST с token-auth и изоляцией по `?company=`

## Архитектурный подход

- модульный монолит (Django + Next.js)
- чёткие границы между доменными модулями
- явная конфигурация по окружениям
- локальная разработка через Docker (PostgreSQL) + venv
- операционный CRM — во frontend; `/admin/` — панель платформы

```
memberships   — абонементы

```

## Зачем такая структура

Продукт рассчитан на долгий рост. Кодовая база должна оставаться понятной новичку и достаточной для коммерческого масштаба. В любой момент должно быть ясно:

1. Что делает этот файл?
2. Зачем он существует?
3. Как безопасно его изменить?

Учебный контур (уроки 13–21) отвечает на вопрос **как добавлять новые фичи** по образцу уже существующих.
