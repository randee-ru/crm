# Этап 3 - SaaS-основа

## Цель

Построить первую настоящую SaaS-основу: компанию, филиал и правила связи между ними.

## Что добавлено

- модель `Company`
- модель `Branch`
- модель `CompanyMembership`
- автогенерация slug
- связи через `ForeignKey`
- тесты моделей
- простая админ-регистрация

## Почему это важно

Если в проекте есть несколько компаний, их данные нельзя смешивать.
Сначала мы явно задаём границы: компания - это tenant, филиал - часть компании.

## Что нужно помнить новичку

- `Company` - это не просто запись в базе, а отдельный клиент платформы
- `Branch` - это подразделение компании
- `CompanyMembership` - это связь пользователя с компанией и ролью
- данные всегда должны идти через связь с компанией

## Как проверить

1. создать компанию
2. создать филиал для этой компании
3. создать пользователя и membership
4. проверить, что slug создаются автоматически
5. проверить, что филиал принадлежит нужной компании
