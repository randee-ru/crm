# Этап 6 - Основа frontend

## Цель

Собрать frontend-каркас CRM Kit, который уже выглядит как продукт и готов подключаться к backend.

## Что добавлено

- главная страница с презентационным блоком
- страница входа
- dashboard-шаблон
- API-клиент для healthcheck
- общие UI-компоненты
- базовая навигация
- пример `.env.example` для адреса backend

## Почему это важно

Frontend не должен начинаться с хаоса из страниц и стилей.
Нужна опорная структура, чтобы дальше быстро собирать продуктовые экраны без постоянной переделки.

## Что нужно помнить новичку

- app shell - это основа всего frontend-приложения
- dashboard - это рабочая панель сотрудников
- API-клиент нужен, чтобы frontend говорил с backend
- переиспользуемые компоненты экономят время и делают UI единым

## Как проверить

1. открыть главную страницу
2. открыть `/login`
3. открыть `/dashboard`
4. убедиться, что dashboard читает `healthcheck`
5. проверить, что навигация и карточки выглядят единообразно

## Связанные файлы

- `frontend/app/page.tsx`
- `frontend/app/login/page.tsx`
- `frontend/app/dashboard/page.tsx`
- `frontend/lib/api.ts`
- `frontend/components/`
- `frontend/.env.example`

## Следующий шаг

После основы frontend можно переходить к автоматизации и уведомлениям или подключать реальные данные из backend к dashboard.
