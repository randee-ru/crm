# Этап 8 — Канбан продаж и воронки CRM

## Цель

Сделать **рабочий канбан сделок** для фитнес-клуба: этапы и воронки хранятся в базе, настраиваются через API, отображаются динамически во frontend.

## Проблема, которую решали

Первая версия канбана использовала захардкоженный enum этапов во frontend и CharField в модели `Deal`. Это не масштабировалось на:

- разные воронки (абонементы, корпоратив, продление);
- настройку этапов без релиза;
- цвета и порядок колонок из админки.

## Результаты

### Backend

- Модели `DealPipeline`, `DealStage`, обновлённый `Deal` с FK на воронку и этап
- Сервис `ensure_default_pipeline()` с дефолтными этапами фитнес-клуба
- Миграция `0003_deal_pipelines` с переносом старых сделок
- API: `/api/v1/pipelines/`, вложенные `/stages/`, фильтр сделок `?pipeline=`
- Тесты в `crm/tests/test_deal_api.py`
- Регистрация в Django admin (при `ADMIN_ENABLE_BUSINESS_MODELS=True`)

### Frontend

- Динамический `CrmKanbanBoard` — колонки из API
- `CrmFunnelSelect` — переключение воронок
- Server actions с `stage_id` и `pipeline_id`
- Раздел настроек «Воронки CRM»
- Параметр URL `?pipeline=<id>`

### Данные

`seed_demo` создаёт 5 демо-сделок в разных этапах дефолтной воронки «Продажи абонементов».

## Дефолтные этапы

| Порядок | Название | Код | Тип |
|---------|----------|-----|-----|
| 10 | Новая заявка | `new_lead` | рабочий |
| 20 | Пробное занятие | `trial` | рабочий |
| 30 | Визит состоялся | `trial_done` | рабочий |
| 40 | Коммерческое предложение | `offer` | рабочий |
| 50 | Ожидает оплату | `payment` | рабочий |
| 60 | Абонемент оформлен | `won` | успех |
| 70 | Отказ | `lost` | проигрыш |

## Критерии готовности

- [x] Этапы канбана читаются из PostgreSQL
- [x] Можно создать вторую воронку через API
- [x] Drag-and-drop сохраняет `stage_id` в БД
- [x] Фильтр сделок по `pipeline`
- [x] Миграция не ломает существующие сделки
- [x] Тесты API проходят
- [x] Документация и урок для разработчиков

## Как проверить локально

```bash
# Backend
cd backend
../.venv/bin/python manage.py migrate --settings=config.settings.dev
../.venv/bin/python manage.py seed_demo --settings=config.settings.dev
../.venv/bin/python manage.py runserver 127.0.0.1:8000 --settings=config.settings.dev

# Frontend
cd frontend
npm run dev
```

Откройте `http://localhost:3000/dashboard` → вкладка «Сделки» → 7 колонок, перетаскивание, селектор воронки.

Логин после seed: `admin` / `121351`.
